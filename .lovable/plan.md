# خطة بناء تطبيق "حلقة القرآن الكريم"

أكمل التطبيق التفاعلي الذي بدأتِه (CSS فقط)، وأملأ كل البيانات الناقصة، وأجهّز إعدادات Android لإنتاج APK لاحقاً.

## النطاق الوظيفي

- **النطاق القرآني:** السور 42 (الشورى) → 57 (الحديد) — 16 سورة
- **الجدول:** 6 أشهر × 4 أسابيع = 24 أسبوع × 3 حصص = 72 حصة
- **مراحل الحصص:** تهيئة / حفظ / مراجعة / إتقان (لون لكل مرحلة)
- **3 تفاسير:** الميسّر، الجلالين، تنوير المقباس (ابن عباس)
- **الصوت:** الشيخ محمد صديق المنشاوي (مرتل)
- **العمل بدون إنترنت** بعد التحميل الأول
- **قابل للتثبيت** على الهاتف (PWA + Capacitor لإنتاج APK)

## مصادر البيانات (مجانية، عامة)

- **نص المصحف + التفاسير الثلاثة:** AlQuran.cloud API
  - نص: `https://api.alquran.cloud/v1/surah/{n}/quran-uthmani`
  - الميسّر: `ar.muyassar` — الجلالين: `ar.jalalayn` — ابن عباس: `ar.miqbas`
- **الصوت (المنشاوي مرتل):** EveryAyah CDN
  - `https://everyayah.com/data/Minshawy_Murattal_128kbps/{sss}{aaa}.mp3`

## بناء جدول الـ24 أسبوع

أوزّع 16 سورة على 24 أسبوع بحيث تنتهي السور الطويلة عبر عدة أسابيع والسور القصيرة تُجمَع. لكل أسبوع 3 حصص:

| الحصة | المرحلة | المحتوى |
|---|---|---|
| 1 | تهيئة (ذهبي) | قراءة المقطع + استماع |
| 2 | حفظ (أخضر) | حفظ آيات الأسبوع |
| 3 | مراجعة (أزرق) | مراجعة + تثبيت |

كل 4 أسابيع تنتهي بـ**حصة إتقان/ختام** (أحمر) للشهر.

## شاشة الحصة (Modal/Route)

- **تبويب المصحف:** عرض الآيات بخط عثماني (Amiri Quran). الضغط على آية → تشغيل صوت + تمييز.
- **زر تشغيل الكل:** يشغّل آيات الحصة بالتتابع.
- **تبويب التفسير:** يبدّل بين الثلاثة من القائمة العلوية.
- روابط سريعة لكل آية مع رقمها.

## العمل offline + شريط التحميل

- زر "تحميل جميع البيانات": يجلب نصوص الـ16 سورة + 3 تفاسير + كل ملفات MP3 للآيات في النطاق (≈ بضع مئات الميجابايت — مع تحذير للمستخدم).
- زر "تحميل الصوتيات فقط".
- شريط تقدم نسبي + حفظ في IndexedDB.
- Service Worker عبر `vite-plugin-pwa` لتخزين الواجهة (App Shell).

## التثبيت كتطبيق هاتف

1. **PWA جاهز للتثبيت** فوراً (manifest + أيقونات + SW).
2. **APK عبر Capacitor:** أضيف `capacitor.config.ts` + سكربتات `cap:add:android` و`cap:build`. بعد ذلك تصدّرين المشروع إلى GitHub وتشغّلين محلياً:
   ```bash
   bun run build
   bunx cap add android
   bunx cap sync
   bunx cap open android   # ثم Build APK من Android Studio
   ```
   (لوفابل لا يبني APK داخلياً، لكن المشروع سيكون جاهزاً 100%.)

## تفاصيل تقنية

- **TanStack Query** للجلب + التخزين مع `staleTime: Infinity` للبيانات القرآنية الثابتة.
- **IndexedDB** عبر `idb-keyval` للتخزين الدائم (نص + JSON تفاسير + blobs صوتية).
- **Tailwind v4** مع توكنز الألوان من الـ HTML الأصلي (`--primary:#1a3d2b`, `--accent:#c9973a`, إلخ).
- **خطوط Amiri / Amiri Quran / Tajawal** عبر `<link>` في `__root.tsx`.
- **RTL** على مستوى `<html dir="rtl" lang="ar">`.
- **Viewport افتراضي:** mobile (التطبيق مصمم للهاتف).

## ما يحتاج تأكيداً منكِ قبل البدء

- **توزيع السور على الأسابيع:** هل تريدين أن أوزّعها بنفسي بشكل متوازن (الشورى 4 أسابيع، الزخرف 3، الدخان 1، الجاثية 1، الأحقاف 2، محمد 1، الفتح 2، الحجرات 1، ق 1، الذاريات 1، الطور 1، النجم 1، القمر 1، الرحمن 1، الواقعة 1, الحديد 2 = 24)؟ أو لديكِ توزيع محدد؟
- **حجم تحميل الصوت كبير** (~300-500MB لكل النطاق). هل أجعله **اختيارياً سورة بسورة** بدل دفعة واحدة؟
